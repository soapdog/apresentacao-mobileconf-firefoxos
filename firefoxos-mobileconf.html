O Que é o Firefox OS?

O Firefox OS é um novo sistema operacional móvel desenvolvido pelo projeto Boot to Gecko (B2G) da Mozilla. Ele usa o Kernel do Linux e inicializa em um sistema baseado no Gecko que permite que os usuários utilizem apps inteiramente desenvolvidos utilizando HTML, Javascript e outras APIs da Web Aberta.

Arquitetura do Firefox OS

Gaia

Coleção de aplicativos web que fazem a UI

Gecko

Motor de JS e Renderização de HTML, APIs amigáveis para acesso ao hardware

Gonk

Kernel Linux, Device drivers e camada de abstração do hardware

App manifest

Você define um arquivo de manifesto que descreve o seu app e diz para o mundo quais icones usar, qual acesso ele precisa e a partir de quais servidores ele pode ser instalado

Instalando apps a partir da web

        var installapp = navigator.mozApps.install(manifestURL);
        installapp.onsuccess = function(data) {
          // App foi instalada
        };
        installapp.onerror = function() {
         // App não foi instalada, informações em 
         // installapp.error.name
        };
Um modo bastante simples de tornar seu app instalável em um aparelho com Firefox OS é utilizando o método install(). Isso significa que você não precisa esperar pela verificação e listagem do marketplace. Voce pode transformar qualquer site em uma app bastando para isso que o usuário abra a página.

Tres níveis de acesso…

  • Apps hospedados - armazenada no seu servidor, fácil de atualizar, acesso ao hardware limitado.
  • Apps privilegiados - verificada pelo Marketplace, utiliza uma Politica de Segurança de Conteúdo, armazenada em um servidor confiável.
  • Apps certificados - parte do sistema operacional, apenas para a Mozilla e parceiros.

Permissões para apps

Aplicativos da Web Aberta vem em tres diferentes versões que permitem maior ou menor acesso ao hardware. Para desenvolvedores, apps hospedadas e privilegiadas são as que podemos construir. Apps hospedadas podem utilizar todos os recursos do HTML5. Apps privilegiadas podem acessar mais funções de hardware.

Utilizando WebAPIs para tornar a web mais capaz

As Web APIs disponíveis para applicativos hospedados ja te levam bem longe. Várias delas já estão disponíveis cross-browser e no Desktop como por exemplo a Battery API. Outras APIs interessantes são push notifications e Vibration para interagir com o usuário.

Web APIs (apps privilegiadas)

  • Device Storage API
  • Browser API
  • TCP Socket API
  • Contacts API

Utilizando WebAPIs para tornar a web mais capaz

Apps privilegiadas podem fazer mais coisas que apps hospedadas pois o servidor é certificado e conhecido de antemão. Por exemplo, você pode realizar chamadas para outros (que suportem CORS) servidores via XHR ou armazenar/recuperar conteúdo do aparelho fora do indexedDB.

Utilizando WebAPIs para tornar a web mais capaz

Aplicativos certificados são aplicativos que fazem parte do sistema operacional e nesse momento só podem ser construídas pela Mozilla e seus parceiros. Eles tem total acesso as funcionalidades do telefone.

Contacts API

          var contact = new mozContact();
          contact.init({name: "Odin"});
          var request = navigator.mozContacts.save(contact);
          request.onsuccess = function() {
            // contato salvo com sucesso
          };
          request.onerror = function() {
            // não foi possível salvar o contato
          };
Para um app privilegiado é fácil criar um novo contato. Isso permite que você sincronize os contatos em vários aparelhos por exemplo.

https://wiki.mozilla.org/WebAPI/WebActivities

Web Activities são uma proposta de padrão para ativar uma interação com o usuário que é parte do sistema operacional. Em vez de utilizar a diretamente a funcionalidade do SO, você usa a mesma via uma interação com o usuário. Dessa forma você não tem problemas de segurança que você normalmente teria se acessasse a funcionalidade diretamente. O Android tem algo parecido chamado "Intents". O Chrome tinha mas foi removido do desktop por causa de UX.

Dando controle para o usuário

Quando iniciamos uma Web Activity, o usuário recebe um aviso para autorizá-la. Por exemplo a atividade "pick" pode pedir imagens da pasta de images, da pasta de papéis de parede ou tirar uma nova foto. Essa é a beleza das atividades - do ponto de vista do desenvolvedor, não importa da onde o conteúdo venha ou para onde ele vai. Decidir sobre isso é responsabilidade do usuário. Bem parecido quando você clica com o botão direito do mouse em um arquivo e escolhe qual programa utilizar para abri-lo.

Enviando um número para o telefone

        var call = new MozActivity({
          name: "dial",
          data: {
            number: "26091048"
          }
        });
Esse é todo o código necessário para enviar um número de telefone para o discador. Tudo que o usuário precisa fazer em seguida é apertar o botão de discar.

Pegando uma imagem do telefone (1/2)

        var getphoto = new MozActivity({
          name: "pick",
          data: {
            type: ["image/png", "image/jpg", "image/jpeg"]
          }
        });
Pegar uma imagem é feito chamando a atividade "pick" e passando os tipos de imagens que você quer.

Pegando uma imagem do telefone (2/2)

        getphoto.onsuccess = function () {
          var img = document.createElement("img");
          if (this.result.blob.type.indexOf("image") != -1) {
            img.src = window.URL.createObjectURL(this.result.blob);
          }
        };
        getphoto.onerror = function () { // erro!
        };
Uma vez invocada, você possui eventos de sucesso e erro e a imagem vem como um blob.
